# ServiceHub — 로컬 서비스 통합 관리 허브

내 PC의 여러 로컬 프로젝트(웹앱·워커)를 **한 곳에서 발견·시작·중지·모니터링**하는 트레이 상주 대시보드입니다.
포트마다 터미널을 따로 띄우던 걸 트레이 아이콘 + `localhost:3050` 대시보드 하나로 묶습니다.

- 지정한 **루트레포 폴더**를 스캔해 그 안의 프로젝트를 **자동 발견**(web/task)
- 대시보드에서 **시작/중지** — 콘솔 창 없이, 종료 시 자식 프로세스까지 정리
- **포트 자동 조율**(충돌 시 다른 포트 재배정) · WebSocket 실시간 상태/로그
- 미니뷰 위젯 · 업무 타이머 · Claude 사용량 표시
- **첫 실행 온보딩** — 처음 열면 "관리할 루트레포 폴더"를 고르는 화면이 떠서, 각자 자기 레포를 지정

> **Windows 전용.** 파이썬 패키지는 설치킷이 자동 설치하지만, **Python 3.11~3.13 은 미리 깔려 있어야 합니다**(없으면 설치 중 안내가 뜹니다).

---

## 🚀 설치 (3단계)

1. **`ServiceHub-kit.zip`** 을 받아 **원하는 위치에 압축 해제**합니다.
   (내 프로젝트들 근처 아무 폴더나 OK — 모노레포 안일 필요 없습니다. TOMY_PUBLIC 안에서 직접 실행하진 마세요.)
2. 풀린 `ServiceHub\` 폴더에서:
   - **비개발자**: **`설치.bat`** 더블클릭 → 의존성 설치·검증 후 트레이로 기동.
   - **Claude Code 사용자**: 그 폴더에서 Claude에게 → `README의 '팀원 로컬 설치' 런북대로 설치해줘.` (설치·기동·온보딩까지 Claude가 대신 처리)
3. 브라우저에서 `http://localhost:3050` → **첫 화면 온보딩**에서 관리할 **루트레포 폴더**를 선택
   (경로 붙여넣기/찾기 → 발견된 폴더 체크 → **시작하기**).

끝. 이후엔 폴더의 `시작.vbs`(콘솔 없이 트레이) 또는 부팅 자동실행으로 뜹니다.

> Python이 없다면: https://www.python.org/downloads/ 에서 **3.11~3.13** 설치(설치 첫 화면 **"Add python.exe to PATH" 체크**) 후 `설치.bat` 재실행.

---

## 🎛 사용법

- **트레이 아이콘**: 작업표시줄 우측 **`^`(숨겨진 아이콘)** 영역. 우클릭 → 미니뷰 표시/숨김 · 대시보드 열기 · 부팅 시 자동 시작 · 설정 파일 열기 · 종료.
- **대시보드**(`localhost:3050`): 서비스 카드에서 시작/중지·로그·폴더/Claude 열기, 헤더에서 다시 스캔·미니뷰 토글·전체 중지.
- **부팅 자동실행**(선택): 트레이 우클릭 → "부팅 시 자동 시작" 토글.

---

## 🗑 제거

풀린 폴더의 **`제거.bat`** 더블클릭 → 부팅 자동실행 해제 + 이 폴더에서 띄운 프로세스 종료.
(소스·설정은 폴더에 남습니다. 완전 삭제는 폴더를 지우세요.)

---

## 🛠 문제 해결 · 자세한 내용

압축 푼 폴더 안의 **`README.md`** 에 전체 런북(팀원 Claude용 자동 설치 절차)·트러블슈팅·구조 설명이 있습니다.

| 증상 | 해결 |
|---|---|
| `설치.bat`이 "Python이 없습니다" | Python 3.11~3.13 설치(Add to PATH) 후 재실행 |
| 대시보드는 뜨는데 서비스가 없음 | 온보딩에서 고른 폴더에 프로젝트가 없음 → 다른/상위 폴더 선택 |
| 온보딩 "찾기"가 안 열림 | 폴더 경로를 왼쪽 입력란에 직접 붙여넣기 |
| 미니뷰만 안 뜸 | WebView2 런타임 미설치 — 대시보드·트레이는 정상. 필요 시 WebView2 Runtime 설치 |
| Windows가 실행을 막음 | 내부 배포라 서명 없음 — "추가 정보 → 실행"으로 허용 |
