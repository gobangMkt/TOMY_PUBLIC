# QA Full Cycle

TC 설계 → 사용자 승인 → Playwright E2E 실행 → 결과 리포트. 단계를 건너뛰지 않는다.

## 워크플로우 (5단계 순서 절대 준수)

---

### 1단계: 분석

테스트 대상을 파악한다. 없으면 묻는다.

- 확인 항목: URL / 기능명 / 요구사항 / 사전 로그인 여부
- 애매한 정보로 TC를 억지로 만들지 말 것. 물어보는 게 낫다.

---

### 2단계: TC 작성

아래 구조로 TC 목록을 만든다. **4가지 유형 모두 포함**이 원칙이나, 기능 복잡도에 따라 생략 가능.

**TC 구조:**

| 필드 | 내용 |
|---|---|
| ID | TC-001 형식, 순번 |
| 제목 | 한 줄 요약 (동사 + 대상) |
| Preconditions | 테스트 시작 전 필요한 상태 |
| Steps | 번호 매긴 액션 (동사로 시작: 클릭, 입력, 이동) |
| Test Data | 실제 사용할 값 (이메일, 비밀번호, 수량 등) |
| Expected Result | 기대 결과 — "성공 메시지 표시"가 아니라 "OOO 텍스트가 화면에 보임"처럼 구체적으로 |
| Postconditions | 테스트 후 시스템 상태 (선택) |

**4가지 유형:**

| 유형 | 목적 | 예시 |
|---|---|---|
| Happy Path | 정상 흐름 | 유효한 이메일/비밀번호로 로그인 성공 |
| Negative Path | 오류 처리 | 잘못된 비밀번호 → 에러 메시지 노출 |
| Edge Case | 경계값 | 255자 최대 입력, 빈 값 제출 |
| Business Logic | 복합 흐름 | 장바구니 추가 → 쿠폰 적용 → 결제 |

---

### 3단계: 사용자 확인 ← **이 단계 없이 실행 절대 금지**

TC 목록을 표로 보여주고 명시적 승인을 받는다.

```
## TC 목록 (총 N개)

| ID | 제목 | 유형 |
|----|------|------|
| TC-001 | ... | Happy Path |
...

승인하시면 E2E 실행을 시작합니다. 수정 사항이 있으면 말씀해주세요.
```

수정 요청 시 반영 후 재확인. 승인이 없으면 4단계로 넘어가지 않는다.

---

### 4단계: E2E 실행 (Playwright MCP)

승인된 TC를 순서대로 실행한다.

**셀렉터 우선순위 — 위에서부터 순서대로 시도:**

1. `getByRole('button', { name: '...' })` — 버튼, 링크, 헤딩
2. `getByLabel('...')` — 폼 필드
3. `getByText('...')` — 텍스트 콘텐츠
4. `locator('[data-testid="..."]')` — 전용 테스트 속성
5. CSS 클래스 — 최후의 수단 (DOM 구조 변경에 취약하므로 지양)

**실행 규칙:**

- 고정 `sleep` 금지 → `waitForSelector`, `waitForNavigation` 등 이벤트 기반 대기 사용
- 각 TC는 독립적으로 실행. 이전 TC의 상태에 의존하지 않는다.
- 실패 시: 스크린샷 캡처 후 다음 TC 계속 진행 (중단하지 않음)
- 재시도: FAIL 판정 전 1회 재시도 허용

**TC 실행 중 출력 형식:**
```
▶ TC-001 실행 중... ✅ PASS
▶ TC-002 실행 중... ❌ FAIL (스크린샷: ./qa-results/TC-002.png)
```

---

### 5단계: 결과 리포트

```markdown
## QA 결과 리포트 — {기능명} ({날짜})

| TC ID | 제목 | 유형 | 결과 | 비고 |
|-------|------|------|------|------|
| TC-001 | 로그인 정상 흐름 | Happy Path | ✅ PASS | |
| TC-002 | 잘못된 비밀번호 | Negative | ❌ FAIL | 에러 메시지 미노출 |

**총 요약:** {N}개 중 {M}개 통과 ({%}%)

### ❌ 실패 상세
- **TC-002**: 예상 — "비밀번호가 올바르지 않습니다" 텍스트 노출 / 실제 — 아무 메시지 없음
  - 스크린샷: ./qa-results/TC-002.png
```

## 빠른 참조

- **TC 없이 E2E 실행** → 금지
- **승인 없이 4단계 진입** → 금지
- **CSS 클래스 셀렉터 우선 사용** → 지양
- **고정 sleep** → 금지
- **실패 시 즉시 중단** → 금지, 계속 진행
