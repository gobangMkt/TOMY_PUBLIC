# 아이콘 디자인 스킬

UI 작업 중 아이콘 요소가 포함된 모든 상황에서 아래 지침을 따른다.

## 적용 시점

- SVG 파일/코드 생성 또는 수정
- UI 컴포넌트(버튼, 카드, 리스트, 배지 등)에 아이콘 추가
- 파비콘, 로고, 앱 아이콘 작업
- 기존 아이콘 교체·리디자인

## 핵심 원칙 (요약)

아이콘 생성 전 `references/icon-style.md`를 반드시 참조한다. 핵심만 요약하면:

| 항목 | 규칙 |
|---|---|
| 스타일 | 100% 플랫 — 그래디언트·그림자 금지 |
| 외곽선 | **stroke 금지** — 모든 요소 fill 전용 |
| viewBox | `0 0 20 20` |
| 컬러 | 5종 이내, 페일·파스텔 금지 |
| 디테일 | 3단계 이하 (Major → Sub → Micro) |

**절대 금지**: Lucide/Heroicons 스타일 stroke outline, 그라데이션, 그림자, glassmorphism

## 작업 절차

1. `references/icon-style.md` 열어 컬러 팔레트·형태 규칙 확인
2. viewBox `0 0 20 20`, fill-only SVG로 작성
3. 컬러는 팔레트 토큰(예: `#FFC84D`, `#EF4452`) 에서만 선택
4. stroke 속성이 있으면 제거, fill 도형으로 대체
5. 완성 후 자기 검토: 그라데이션·그림자·stroke 없는지 확인

## 참조

- 전체 스펙: `references/icon-style.md`
